

# إصلاح مشكلتين: خطأ البناء + أرشيف التسوية

## المشكلة 1: خطأ البناء المتكرر
`vite` **غير موجود** في `devDependencies` بملف `package.json` (السطر 86-93). رغم المحاولات السابقة، لم تتم إضافته فعلياً.

النتيجة: `bun x vite@5.4.19` يجد نسخة تالفة من vite في node_modules (تبعية فرعية لـ `@vitejs/plugin-react`) ويستخدمها → ملف `dep-C6uTJdX2.js` مفقود.

### الحل
إضافة `"vite": "5.4.19"` في `devDependencies` بملف `package.json`:

```json
"devDependencies": {
    "@types/node": "^24.1.0",
    "@types/react": "^18.3.23",
    "@types/react-dom": "^18.3.7",
    "eslint": "^8.57.1",
    "eslint-config-react-app": "^7.0.1",
    "vite": "5.4.19",
    "vite-plugin-pwa": "^0.20.5"
}
```

## المشكلة 2: أرشيف التسوية لا يعرض الطلبات
بعد فحص الكود، المنطق في `EmployeeFollowUpPage.jsx` **صحيح بالفعل**:
- السطر 573-574: طلبات `no_rule_archived/no_rule_settled` تمر عند `showSettlementArchive`
- السطر 631-633: `isSettled` يشمل الحالات الثلاث
- السطر 949-952: العداد يحسب الحالات الثلاث

**السبب الحقيقي**: خطأ البناء يمنع نشر التغييرات! الكود المعدّل لم يُبنى ولم يُنشر أبداً بسبب فشل البناء المتكرر.

بمجرد إصلاح خطأ البناء، ستعمل تغييرات أرشيف التسوية تلقائياً.
