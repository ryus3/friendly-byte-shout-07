## رابعاً: تدفّق العمل الكامل للـ Off-Channel

### 1) لحظة استلام فاتورة الوسيط
عند تأكيد الاستلام، النظام يفحص تلقائياً كل طلب فيها:
- إذا مبلغ الطلب من الوسيط = 0 **AND** تم تسليمه (`delivery_status = 4`) **AND** غير راجع/تبديل:
  - يُدرَج في جدول جديد `off_channel_collections` بحالة `pending_classification`.
  - يظهر **شارة حمراء** على كارت "تحصيلات خارج القناة" تطلب التصنيف.

### 2) نافذة التصنيف (تفتح بنقرة على الطلب)
الحقول:
- **نوع التحصيل** (إجباري — قائمة منسدلة):
  - 💳 دفع إلكتروني (زين كاش / ماستر / آسيا)
  - 🏦 تحويل بنكي
  - 💵 الموظف قبض نقداً من الزبون
  - 🎁 خصم كامل للزبون (لا يوجد مبلغ مستحق)
  - 🚚 المالك يتحمّل التوصيل فقط (المنتج هدية/تعويض)

- **المبلغ المقبوض من الزبون** (إجباري — قابل للتعديل):
  - افتراضياً = (سعر المنتجات + أجور التوصيل)
  - الموظف يدخل القيمة الفعلية. مثلاً لو الزبون دفع السعر كاملاً مع التوصيل = 30,000.
  - لو دفع السعر فقط = 25,000.
  - لو دفع جزء فقط = أيّ رقم.

- **من قبض المبلغ؟** (إجباري):
  - الموظف منشئ الطلب (افتراضياً)
  - المالك مباشرة
  - طرف آخر (مع كتابة الاسم)

- **ملاحظة** (اختيارية): رقم العملية، اسم البنك، إلخ.

### 3) ما يحدث بعد التصنيف (حسب الحالة)

#### الحالة A: الموظف قبض المبلغ وله قاعدة ربح (مثال: ربحه 7,000)
```
سعر المنتج:           25,000
أجور التوصيل:          5,000
المبلغ المقبوض:        30,000  (سعر + توصيل)

دَيْن الموظف للمالك = المقبوض − ربح الموظف
                  = 30,000 − 7,000
                  = 23,000   ← يُضاف لـ employee_debts
ربح الموظف:           7,000   ← يبقى للموظف (لا يدخل employee_dues لأنه مقبوض فعلاً)
ربح المالك المحاسبي:  25,000 − 10,000 (تكلفة) − 7,000 (ربح موظف) = 8,000
الخسارة على القناة:   −5,000 (أجور التوصيل، يتحمّلها المالك حتى يدفع الموظف)
```

#### الحالة B: الموظف قبض المبلغ ولا قاعدة ربح له (موظف عادي/متطوّع)
```
دَيْن الموظف للمالك = المبلغ المقبوض كاملاً = 30,000
ربح المالك: السعر − التكلفة = 15,000
```

#### الحالة C: المالك/المدير هو منشئ الطلب
```
لا دَيْن — المبلغ ذهب للمالك مباشرة.
يُسجَّل قيد "Off-channel Receipt" في القاصة الرئيسية بقيمة المقبوض.
ربح المالك يُحسب طبيعياً.
```

#### الحالة D: خصم كامل (هدية/تعويض)
```
لا دَيْن، لا إيراد.
COGS + أجور التوصيل تُسجَّل كـ "مصاريف ترويج/تعويض زبون".
لا ربح للموظف ولا للمالك (الطلب خسارة كاملة محسوبة).
```

### 4) تأكيد المالك (الخطوة الذهبية لإقفال الدَّيْن)
بعد أن يصنّف الموظف، يظهر السجل عند المالك في صفحة جديدة **"تحصيلات بانتظار تأكيدي"**:
- يرى: الطلب + الموظف + المبلغ المتوقَّع.
- زرّان: ✅ **استلمت المبلغ** / ⏸️ **لم يصلني بعد**.
- عند الضغط على "استلمت":
  - حركة نقد `+المبلغ` في القاصة الرئيسية، نوع `off_channel_receipt`.
  - `employee_debts` تُحدَّث (خصم من الدَّيْن).
  - حالة السجل → `settled`.
  - إشعار للموظف "تم تأكيد استلام مبلغك".

### 5) المقاصّة مع مستحقات الموظف
عند تسوية مستحقات الموظف الشهرية:
```
مستحقات الموظف = Σ ربحه من الطلبات المُسلَّمة
خصومات       = Σ دَيْن off-channel غير المسوّى
الصافي       = مستحقات − خصومات
```
- لو الصافي موجب → يُدفع للموظف.
- لو سالب → الموظف يدفع للمالك (تحويل/نقد) ويُسجَّل تأكيد المالك.

## سابعاً: قراري النهائي — كل دينار تحت السيطرة

| السؤال | الجواب |
|---|---|
| كيف نعرف أن الموظف حوّل المبلغ؟ | **لا نصدّق إعلانه وحده.** المالك يضغط "استلمت" → عندها فقط يُقفل الدَّيْن. |
| المبلغ مع التوصيل أم بدون؟ | الموظف يُدخل **المبلغ الفعلي الذي قبضه** (أيّاً كان). النظام يحسب الباقي تلقائياً. |
| موظف بقاعدة ربح؟ | يخصم ربحه من الدَّيْن. يدفع للمالك = (المقبوض − ربحه). |
| موظف بدون قاعدة؟ | يدفع كامل المقبوض للمالك. |
| المالك/المدير منشئ الطلب؟ | لا دَيْن، يُسجَّل قيد إيراد off-channel مباشرة بعد تأكيده. |
| ماذا لو الموظف رفض الدفع؟ | الدَّيْن يبقى مفتوحاً ويُخصم من أيّ مستحقات مستقبلية تلقائياً. |
| ماذا لو الزبون لم يدفع شيئاً (هدية)؟ | تصنيف "خصم كامل" → COGS+توصيل كمصروف، لا دَيْن لا ربح. |
