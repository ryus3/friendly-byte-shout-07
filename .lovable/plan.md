سأصلح المشكلة من جذورها بحيث تظهر لك قائمة مناطق فعلية داخل نافذة المساعد، وتختار المنطقة بضغطة واحدة ثم يكمل إنشاء الطلب مباشرة بدون أن تعيد كتابة الطلب مرة ثانية.

1. سبب المشكلة الحالي
- الدالة `ai-gemini-chat` ترجع فعلاً نوع رد اسمه `region_clarification` ومعه `suggestions`.
- لكن الواجهة `SuperAiChatDialog.jsx` لا تتعامل مع هذا النوع نهائياً، بل تعرض النص فقط كرسالة عادية.
- لذلك يظهر لك: “أعد إرسال الطلب مع الاسم الكامل للمنطقة” بدل قائمة أزرار.

2. ما سأبنيه
- إضافة واجهة اختيار مناطق داخل `SuperAiChatDialog.jsx`.
- عندما يرجع الرد `region_clarification`:
  - تظهر بطاقة داخل المحادثة فيها أسماء المناطق المقترحة كأزرار/خيارات واضحة.
  - الضغط على اسم المنطقة يرسل اختيارها مباشرة للنظام.
  - النظام يعيد استخدام نص الطلب الأصلي مع المنطقة المختارة ويكمل إنشاء الطلب تلقائياً.
- لن أستخدم أسلوب “اكتب الاسم الكامل مرة أخرى” إلا كحل احتياطي فقط إذا لم توجد خيارات.

3. كيف سيعمل التدفق الجديد
```text
المستخدم يرسل الطلب
-> ai-gemini-chat يحلل المدينة والمنطقة
-> إذا المنطقة غير محسومة:
   يرجع region_clarification + suggestions + بيانات الطلب المعلقة
-> الواجهة تعرض قائمة أزرار للمناطق
-> المستخدم يضغط المنطقة الصحيحة
-> الواجهة ترسل اختيار المنطقة للدالة
-> الدالة تثبت city_id / region_id الصحيحين
-> يتم إنشاء الطلب مباشرة
```

4. التعديلات المطلوبة
- `supabase/functions/ai-gemini-chat/index.ts`
  - توسيع رد `region_clarification` ليحمل كل ما يلزم لإكمال الطلب لاحقاً بدون فقدان السياق:
    - `original_message`
    - `city_name`
    - `city_id`
    - قائمة المناطق المقترحة بالاسم والمعرف الخارجي الصحيح
    - معرف/رمز متابعة مؤقت أو payload واضح لإعادة الإرسال
  - إضافة مسار جديد لمعالجة “اختيار منطقة من القائمة” بحيث يستخدم المعرف الخارجي الصحيح ويكمل `process_telegram_order`.

- `src/components/ai/SuperAiChatDialog.jsx`
  - إضافة معالجة صريحة لـ `data.type === 'region_clarification'`.
  - تخزين حالة التوضيح المعلقة داخل المحادثة.
  - عرض قائمة خيارات قابلة للنقر بدل الاكتفاء بالنص.
  - عند الضغط على خيار، استدعاء `ai-gemini-chat` مرة ثانية بصيغة “confirm_region_selection”.
  - بعد نجاح الإنشاء، استبدال بطاقة الاختيار برسالة نجاح واضحة.

5. ضمان الدقة
- الأزرار ستعتمد على `alwaseet_id`/المعرف الخارجي الصحيح للمنطقة، وليس مجرد الاسم النصي.
- لن يُنشأ الطلب إذا كانت المنطقة ما زالت غير محسومة.
- سيتم الحفاظ على:
  - `customer_city` = المدينة فقط
  - `customer_province` = المنطقة الصحيحة فقط
  - `customer_address` = النقطة الدالة فقط

6. تحسين تجربة الاستخدام
- عرض عنوان واضح مثل: “اختر المنطقة الصحيحة”.
- إظهار المدينة أعلى القائمة.
- تعطيل الأزرار بعد الاختيار لمنع الضغط المكرر.
- في حال فشل إنشاء الطلب بعد الاختيار، تبقى القائمة موجودة مع رسالة خطأ مفهومة.

## التفاصيل التقنية
- الواجهة الحالية لا تحتوي أي renderer لـ `suggestions`، لذلك أصل المشكلة frontend وليست فقط backend.
- الـ Telegram bot لديه منطق اختيار تفاعلي، لكن `SuperAiChatDialog.jsx` لا يملك equivalent UI له.
- سيتم اتباع نفس فكرة التوضيح الموجودة في التليغرام لكن بصيغة أزرار داخل الشات.
- إذا احتجت حفظ حالة معلقة بين الرسائل، سأستخدم payload منظم من الدالة أو تخزين مؤقت آمن بدل الاعتماد على إعادة كتابة المستخدم للنص.

بعد الموافقة، أنفذ هذا مباشرة.
