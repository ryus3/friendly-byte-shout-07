
# 🎯 خطة الإصلاح الشامل + التسريع

## 🔴 الدفعة A — إصلاحات حرجة

### A1. إصلاح "المناطق = 0" في كاش مدن
**السبب الجذري المُكتشف من logs**: 
- جدول `regions_master` لديه `id` و `alwaseet_id` كحقول `NOT NULL` بدون قيمة افتراضية
- عند `INSERT` من `update-modon-cache`، يفشل لأن الكود لا يمرر هذه الحقول
- نتيجة: 18 مدينة تُسجَّل، لكن **0 منطقة جديدة** (الموجودة 6259 من الوسيط فقط)

**الحل (Migration)**:
```sql
-- جعل id auto-increment
ALTER TABLE regions_master 
  ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY;

-- جعل alwaseet_id nullable (مدن لا تستخدمه)
ALTER TABLE regions_master 
  ALTER COLUMN alwaseet_id DROP NOT NULL;

-- نفس الشيء لـ cities_master احتياطاً
ALTER TABLE cities_master 
  ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY;
```

**ملاحظة**: المعرف الحقيقي للوسيط محفوظ في `city_delivery_mappings.external_id` و`region_delivery_mappings.external_id` — لذلك حذف `alwaseet_id` من master آمن تماماً.

### A2. إصلاح خطأ "ON CONFLICT DO UPDATE command cannot affect row a second time"
في `update-modon-cache` السطر 76-81 من logs، هناك تكرار في mappings (نفس `region_id` يظهر مرتين). 
**الحل**: deduplicate قبل upsert في `upsertRegionsBatch`.

### A3. إصلاح errNum:21 في الوسيط (صلاحية الوصول)
`errNum:21` من API الوسيط = **التوكن منتهي/غير صالح** (ليس IP).

**الحل في `alwaseet-proxy/index.ts`**:
- بعد parse الـresponse، إذا `data.errNum === 21` أو `data.errNum === '21'`:
  - تسجيل في log واضح
  - إرجاع رسالة عربية واضحة: "انتهت جلسة الوسيط، يرجى تسجيل الدخول مجدداً"
  - في `AlWaseetContext`، عند استلام `errNum:21` → مسح localStorage + إظهار dialog إعادة تسجيل الدخول تلقائياً

**تنظيف DB**: حذف صف `alshmry94` المنتهي (المكرر، يوجد نسخة صالحة)، وصف `lab@ryus-brand` المنتهي.

### A4. توحيد طبقة الترجمة (المدن/المناطق)
الهيكلة الموحدة موجودة بالفعل:
- `cities_master` / `regions_master` = "Hub" داخلي موحد بأسماء عربية
- `city_delivery_mappings(city_id, delivery_partner, external_id, external_name)` = جسر لكل شركة
- `region_delivery_mappings(region_id, delivery_partner, external_id, external_name)` = نفس الفكرة

بعد إصلاح A1+A2: 
- ✅ مدن سترسل المناطق الصحيحة → `regions_master` → `region_delivery_mappings(partner='modon')`  
- ✅ ai-gemini-chat (بوت التليغرام) يستخدم `cities_master`/`regions_master` ⇒ يفهم العناوين تلقائياً لكلتا الشركتين
- ✅ أي شركة جديدة مستقبلاً = نفس النمط (delivery_partner='shركة_جديدة')
