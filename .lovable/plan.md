
## فهم قواعد التوريث في الطلبات المركبة (`+`)

### القاعدة الذهبية الموحّدة
عند تقسيم النص بـ `+`، لكل جزء بعد الأول، يتم **التوريث الذكي الثلاثي**:

| العنصر | متى يُورَّث من الجزء السابق؟ |
|---|---|
| **المنتج** | إذا لم يُذكر اسم منتج في الجزء الحالي |
| **اللون** | إذا لم يُذكر لون في الجزء الحالي |
| **القياس** | إذا لم يُذكر قياس في الجزء الحالي |

ما يُذكر صراحة في الجزء الحالي **يحلّ محل** الموروث.

### الأمثلة التي وضحتها (مؤكدة)

| النص | الجزء 1 | الجزء 2 | منطق التوريث |
|---|---|---|---|
| `نايك نيلي سمول + ميديم` | نايك / نيلي / سمول | نايك / نيلي / **ميديم** | ورّث المنتج + اللون، القياس صريح |
| `نايك نيلي سمول + اخضر` | نايك / نيلي / سمول | نايك / **اخضر** / سمول | ورّث المنتج + القياس، اللون صريح |
| `نايك نيلي + مكسيكي اخضر سمول` | نايك / نيلي / (—) | **مكسيكي / اخضر / سمول** | منتج جديد صريح، لا توريث |
| `نايك احمر ميديم + اخضر سمول` | نايك / احمر / ميديم | نايك / **اخضر** / **سمول** | ورّث المنتج فقط |

### حالات إضافية مدعومة بنفس القاعدة (أوضحها لك للتطبيق)

| النص | النتيجة المتوقعة |
|---|---|
| `نايك احمر سمول + ميديم + لارج` | 3 عناصر: نايك/احمر/S، نايك/احمر/M، نايك/احمر/L |
| `نايك احمر سمول + اخضر + ازرق` | 3 عناصر: نايك/احمر/S، نايك/اخضر/S، نايك/ازرق/S |
| `نايك احمر سمول + اخضر ميديم` | نايك/احمر/S + نايك/اخضر/M |
| `نايك احمر سمول + اخضر ميديم + لارج` | الثالث يرث من الثاني: نايك/اخضر/L |
| `نايك احمر + برشلونة` | نايك/احمر + برشلونة (منتج جديد، لا توريث للون) |
| `نايك احمر سمول + برشلونة ازرق + ميديم` | الثالث يرث من الثاني: برشلونة/ازرق/M |
| `نايك احمر سمول + 2 ميديم` | نايك/احمر/S ×1 + نايك/احمر/M ×2 (الكمية لا تُورَّث) |
| `نايك سمول + ميديم` (بدون لون) | نايك/-/S + نايك/-/M (اللون "-" الافتراضي) |

### قواعد الحماية (لمنع التخريب)
1. **منتج جديد يُصفّر السياق**: عند ذكر منتج جديد صراحة، لا يُورَّث لون أو قياس من قبله (لأن الألوان/القياسات قد لا تكون متوفرة في المنتج الجديد).
2. **التحقق من المخزون لكل جزء مستقلاً**: كل عنصر ناتج يُفحَص ضد جدول `inventory` كما هو الآن.
3. **رسائل البدائل الذكية تبقى تعمل**: إذا كان قياس/لون غير متوفر، تُقترح البدائل لذلك العنصر تحديداً.
4. **الكمية لا تُورَّث أبداً**: الافتراضي 1 لكل عنصر.
5. **الطلب المفرد بدون `+`** يبقى يعمل بنفس المنطق الحالي 100%.

## التشخيص المؤكد للعطل الحالي

اختبرت `extract_product_items_from_text` على قاعدة البيانات:
- ✅ `نايك نيلي سمول` → يعمل
- ❌ `نايك نيلي سمول + ميديم` → "حدث خطأ في معالجة طلبك"

**السبب الجذري**: الدالة تستخدم متغيرات من نوع `record` للتوريث بين الأجزاء، وعند محاولة قراءة حقل من record فارغ يُرمى استثناء يبتلعه `EXCEPTION WHEN OTHERS` ويُرجع رسالة عامة.

## ما سأنفذه (مرحلة A + B + C)

### A. إصلاح اكتمال شريط التقدم في السبلاش
**الملف:** `src/components/AppSplashScreen.jsx` (السطر 39-46)

- لا تغيير في التصميم/الأنيميشن/الألوان/المدة الكلية.

### B. إزالة لودر "Loader2" بعد السبلاش
**الملف:** `src/pages/Dashboard.jsx` (السطر 752 تقريباً)
- استبدال `<Loader2 />` بخلفية شفافة `<div className="h-full w-full bg-background" />`.
- البطاقات تظهر فور جاهزية البيانات (Phase 1 جاهز قبل السبلاش، Phase 2 يستمر بالخلفية).
- النتيجة: انتقال سلس مباشر سبلاش → رئيسية.

### C. إصلاح وتوسيع الطلبات المركبة في بوت تليغرام
**Migration واحدة جراحية** على دالة `extract_product_items_from_text`:

#### التغييرات
1. **استبدال `record` بمتغيرات سكالار آمنة**:
   - `v_inherited_product_id uuid`, `v_inherited_product_name text`
   - `v_inherited_color_id uuid`, `v_inherited_color_name text`
   - `v_inherited_size_id uuid`, `v_inherited_size_name text`
   - يحل الاستثناء جذرياً.

2. **منطق التوريث الثلاثي الذكي**:
   ```
   لكل جزء بعد الأول:
     - ابحث عن منتج في الجزء → إن لم يوجد، استخدم v_inherited_product_*
     - إن وُجد منتج جديد: صفّر اللون والقياس الموروثين (سياق جديد)
     - ابحث عن لون → إن لم يوجد، استخدم v_inherited_color_* (إن لم يكن المنتج جديد)
     - ابحث عن قياس → إن لم يوجد، استخدم v_inherited_size_* (إن لم يكن المنتج جديد)
     - بعد المعالجة: حدّث متغيرات التوريث بقيم الجزء الحالي للجزء التالي
   ```

3. **تحسين معالجة الأخطاء**:
   - استبدال `EXCEPTION WHEN OTHERS THEN RETURN '...عام'` بمعالجة تُرجع رسالة تشخيصية واضحة لكل نوع خطأ، مع الإبقاء على الأمان.

4. **الحفاظ التام على**:
   - توقيع الدالة `(text, jsonb) → jsonb` واسمها.
   - منطق البحث عن المنتجات (3→2→1 كلمة) دون تغيير.
   - تطبيع القياسات (سمول→S، ميديم→M، لارج→L، اكس لارج→XL، …).
   - تطبيع الألوان والمرادفات.
   - فحص المخزون والكميات.
   - رسائل البدائل الذكية.
   - استخدام "-" كقيمة افتراضية للون/القياس.
   - منطق `process_telegram_order` ودوال أخرى (لن تُمَس).

5. **لن تُمَس مطلقاً**:
   - أي جدول، فهرس، RLS، trigger.
   - ملفات `supabase/functions/telegram-bot/*.ts`.
   - الكاش، الإشعارات، Realtime.

#### اختبارات ما بعد التنفيذ (سأشغّلها مباشرة)
| النص | المتوقع |
|---|---|
| `نايك نيلي سمول` | عنصر واحد ✓ (regression) |
| `نايك نيلي سمول + ميديم` | 2 عناصر بنفس المنتج/اللون |
| `نايك نيلي سمول + اخضر` | 2 عناصر بنفس المنتج/القياس |
| `نايك نيلي + مكسيكي اخضر سمول` | 2 عناصر من منتجين مختلفين |
